# Инструкции для Cline

## Общие правила
1. Не вставляй в код комментарии на русском языке.
2. Не используй теги `<...>` в коде.
3. Все изменения в коде должны быть минимальными и только по делу.
4. Перед изменением файла покажи, что именно собираешься менять.

## Проект ENRG Protocol
- Стек: Rust/Anchor (смарт-контракты), Node.js (оракул), ESP32 (прошивка).
- База данных: SQLite (файл `enrg.db` в корне проекта).
- Защита от повторных nonce: хранится в таблице `energy_store`.
- Ключ основателя: через переменную окружения `FOUNDER_KEY` или `FOUNDER_KEYPAIR_PATH`.
- Оракул должен запускаться командой `node server.js`.

## Требования к качеству
- Код должен компилироваться без ошибок (cargo build, cargo clippy).
- Все тесты должны проходить.
- Ошибки обрабатываются явно (без `console.log` в продакшене).
- Секреты не захардкожены.
