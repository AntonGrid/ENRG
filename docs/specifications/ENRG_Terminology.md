# ENRG Terminology

## Status

Normative

---

## Purpose

This document defines the official terminology of the ENRG Protocol.

Every normative term SHALL have exactly one official definition.

Implementations, specifications, SDKs, APIs, and documentation SHALL use the terminology defined in this document.

---

# A

## Audit Log

An immutable chronological record of protocol events.

---

# C

## Capability

A standardized protocol feature identified through the Capability Registry.

## Client Application

Any software implementing ENRG Protocol interfaces for human interaction.

Examples include Web Clients, Mobile Clients, Desktop Clients, and CLI Clients.

---

# D

## Device

A physical system implementing the ENRG Protocol.

## Device Identity

The unique cryptographic identity assigned to a Device.

## Device Lifecycle

The sequence of protocol-defined Device states.

## Device Manifest

A structured document describing Device capabilities and metadata.

## Device Registry

The authoritative source of Device state.

---

# E

## ENRG Network

A network of interoperable implementations of the ENRG Protocol.

## ENRG Protocol

An open, implementation-independent protocol for cryptographically verifiable physical infrastructure.

## Event

A standardized notification describing a completed protocol action.

---

# G

## Governance

The process through which the ENRG Protocol evolves.

---

# M

## Manifest

See Device Manifest.

---

# O

## Oracle Implementation

A compliant implementation responsible for cryptographic verification defined by this specification.

Oracle Implementations SHALL NOT evaluate protocol policy.

---

# P

## Policy Engine

The protocol component responsible for evaluating protocol policy.

## Proof-of-Production (PoP)

Cryptographically verifiable evidence of energy production.

## Provisioning

The process through which a Device receives its operational configuration.

## Protocol Economics

The economic rules governing issuance and circulation of SRC.

---

# R

## Registry

An authoritative protocol data source.

---

# S

## Smart Contract

The authoritative on-chain execution component of the ENRG Protocol.

## SRC

The native protocol token of the ENRG Protocol.

SRC is issued according to the Protocol Economics defined by this specification.

---

# V

## Verification

The process of confirming cryptographic authenticity and protocol compliance.

---

# Normative Language

The key words SHALL, MUST, SHOULD, MAY, and MUST NOT are to be interpreted as described in RFC 2119 and RFC 8174.
