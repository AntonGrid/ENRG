# Overview

This chapter defines the normative and informative references used throughout the ENRG Protocol.

Normative references define mandatory requirements.

Informative references provide additional implementation guidance.

# Reference Implementations

Reference implementations MAY include:

- Smart Contract Implementation
- Oracle Implementation
- ESP32 Firmware
- Reference SDK
- Client Applications

Reference implementations demonstrate protocol compliance but SHALL NOT define protocol behavior.

---

# Protocol Authority

The ENRG Protocol Specification SHALL remain the authoritative definition of protocol behavior.

If any implementation conflicts with this specification, this specification SHALL take precedence.

---

# Version History

Major protocol revisions SHALL be documented through future protocol versions.

Previous revisions SHOULD remain publicly available.

---

# Final Statement

The ENRG Protocol is an open, implementation-independent standard for cryptographically verifiable physical infrastructure.

The protocol is governed.

The protocol is not owned.
