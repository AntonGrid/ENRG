# ADR-0002: Device Registry — единственный источник истины о состоянии устройства

## Статус
Принято

## Контекст
В системе ENRG Protocol состояние устройства (активно, в карантине, обслуживание и т.д.) может храниться в нескольких местах: Oracle, SQLite, кэш, Dashboard. Это создаёт риск рассинхронизации и усложняет диагностику.

## Решение
Device Registry является единственным источником истины для состояния устройства. Все компоненты (Oracle, Dashboard, Provisioning Service) обращаются к Registry для получения актуального состояния. Registry хранит:
- Текущее состояние устройства.
- Владельца (wallet).
- Trust Level.
- Capabilities.
- Время последнего heartbeat.
- Версию прошивки.
- Историю состояний (Audit Log).

## Обоснование
- Единая точка управления состоянием упрощает диагностику и аудит.
- Устраняет риск рассинхронизации между компонентами.
- Позволяет легко масштабировать систему: новые компоненты просто обращаются к Registry.
- Обеспечивает целостность данных.

## Последствия
- Oracle не хранит состояние устройства — он только проверяет подписи и вызывает mint.
- Policy Engine принимает решения на основе данных из Registry.
- Dashboard отображает состояние, полученное из Registry.
- Registry должен иметь высокую доступность (репликация, бэкапы).

## Альтернативы
- Хранение состояния в каждом компоненте отдельно — отклонено из-за риска рассинхронизации.
- Использование блокчейна как источника истины — отклонено из-за задержек и стоимости.
