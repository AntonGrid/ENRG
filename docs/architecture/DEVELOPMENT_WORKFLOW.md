# ENRG Development Workflow

Status: Active

---

# Purpose

This document defines the official development workflow for the ENRG ecosystem.

Every change SHALL follow this workflow.

---

# Stage 1 — Idea

A new concept, feature or protocol improvement is proposed.

Output:

- Idea
- Problem Statement

---

# Stage 2 — Architecture

The proposal is analyzed.

If accepted:

- ADR is created if architectural.
- RFC is created if discussion is required.

Output:

- Approved architecture.

---

# Stage 3 — Protocol Specification

The protocol documentation is updated.

Affected documents may include:

- Protocol Specification
- Registries
- Terminology
- Conformance
- References

The specification becomes the source of truth.

---

# Stage 4 — Roadmap

Implementation tasks are planned.

Output:

- Technical roadmap
- Migration plan
- Compatibility notes

---

# Stage 5 — Implementation

Implementations are updated.

Examples:

- Smart Contract
- Oracle
- SDK
- Dashboard
- Firmware
- Mobile Client

Implementation MUST follow the specification.

---

# Stage 6 — Testing

Implementation is verified.

Includes:

- Unit Tests
- Integration Tests
- Protocol Validation

---

# Stage 7 — Conformance

Implementation is checked against the specification.

Requirements:

- No protocol deviations
- No undocumented behavior
- No incompatible changes

---

# Stage 8 — Release

Implementation becomes part of the ENRG ecosystem.

Documentation and implementation remain synchronized.

---

# Rule

Specification First.

Implementation Second.

Always.
