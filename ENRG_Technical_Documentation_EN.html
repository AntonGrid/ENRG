ENRG Protocol – Technical Documentation v3.0

DePIN / Cyber‑Physical Infrastructure

1. Architecture

IoT Devices (ESP32, industrial Modbus meters) → Oracles (Switchboard) → Smart Contracts (Solana) → Energy Vault ↔ Users / Markets

2. Proof-of-Production (PoP)

Every 10 min, controller sends signed proof {device_id, timestamp, energy_wh, signature}. Contract verifies signature, timestamp window (≤15 min), power limit, nonce, then mints SRC with 15% protocol fee distribution.

3. Industrial Integration

Support for Modbus RTU/RS485 meters (Schneider, Siemens, ABB, Меркурий, Энергомера). Gateway firmware translates Modbus readings into signed proofs.

4. Anti-Fraud System

  • Hardware security: ATECC608 Secure Element (key storage, signing).
  • Replay protection: Monotonic nonce per device.
  • Physical limits: max_power_w per device, time-based limits (10 min).
  • Multi-oracle verification: Switchboard aggregates data from multiple oracles.
  • Anomaly detection: Monitoring sudden spikes, comparison with historical data.
  • Tamper detection: Enclosure sensors → key revocation.
